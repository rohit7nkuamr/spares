Hybrid Database & Data Ingestion Strategy

The **Hybrid Database Strategy** is a critical design choice, separating low-volume transactional data from high-volume sensor readings to ensure long-term performance and scalability.

Data Types & Storage

  • **Relational Data (PostgreSQL):** For structured, transactional data like Equipment Master Registry and Maintenance Logs.
  • **Time-Series Data (InfluxDB/TimescaleDB):** For high-volume, high-velocity time-stamped data from **Future IoT Sensors** (temp, vibration).

Data Ingestion Methods

  • **Historical Data:** Bulk uploads via secure web interface (CSV/Excel).
  • **Manual Data:** Technicians log maintenance/spares usage directly via the Mobile App.
  • **Future IoT Data:** Secure, scalable **MQTT Broker** and REST API endpoints for real-time sensor streams.

MLOps (Model Lifecycle Management)

MLOps is a foundational, non-negotiable component for managing the **dozens of independent models** required for diverse vehicle fleets (e.g., ALS Engine vs. T-90 Transmission).

Core MLOps Components (MLflow)

  • **Experiment Tracking:** Logs all training runs, parameters, and metrics.
  • **Model Registry:** Central repository to store, version, and stage trained models.
  • **Automated Pipelines (CI/CD):** Triggers model retraining when new data is available or performance degrades (Model Drift).

Model Deployment & Monitoring

  • **Serving:** Models are containerized and deployed as microservices with a REST API endpoint for inference.
  • **Monitoring:** Dedicated dashboard tracks real-world model performance (Accuracy, Precision, Recall) to ensure models remain relevant.

Critical Dependencies and Risk Mitigation

The project's success is tied to organizational collaboration and data quality. We have defined three critical dependencies and their mitigation strategies.

Dependency 1: Data Quality & Access

The AI model accuracy depends entirely on the completeness and consistency of historical maintenance logs.

Mitigation Strategy:
  • Mandatory joint data-scoping workshop (Week 2).
  • Co-create explicit Data Requirements Document (DRD).
  • Build automated data validation scripts into the ingestion pipeline.

Dependency 2: Consistent Army SME Access

Expert EME/Ordnance guidance is critical for feature engineering and practical validation of maintenance triggers.

Mitigation Strategy:
  • Formally assign a Primary and Secondary SME.
  • Establish a fixed schedule of interactions (min. 4 hours/week).
  • Utilize collaborative tools for asynchronous feedback.

Dependency 3: UAT/Pilot Unit Availability

Availability of personnel and equipment at 474 EME Wksp is critical for the UAT and training window (Weeks 17-20).

Mitigation Strategy:
  • Schedule UAT/training periods 2+ months in advance with Command.
  • Develop detailed UAT plan for maximum time efficiency.
  • Build a one-week buffer into the project plan.
